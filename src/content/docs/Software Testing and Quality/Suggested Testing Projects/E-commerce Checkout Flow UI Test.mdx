## Goal

Automate a critical user flow:

- browse product
- add to cart
- checkout

## Requirements

- use explicit waits
- use stable selectors (`data-testid`)
- implement Page Objects

## Suggested structure

- `pages/` (page objects)
- `tests/` (test scenarios)

## CI tip

Run UI tests in headless mode and store screenshots on failure.
