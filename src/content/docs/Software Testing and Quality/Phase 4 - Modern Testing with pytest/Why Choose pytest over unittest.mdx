## Key differences

### unittest

- class-based (`unittest.TestCase`)
- assertion methods (`self.assertEqual`)
- more boilerplate

### pytest

- simple functions
- plain `assert`
- powerful fixtures
- rich plugin ecosystem

## Example comparison

### unittest

```python title="test_add_unittest.py" showLineNumbers{1}
import unittest


def add(a, b):
    return a + b


class TestAdd(unittest.TestCase):
    def test_add(self):
        self.assertEqual(add(2, 3), 5)
```

### pytest

```python title="test_add_pytest.py" showLineNumbers{1}

def add(a, b):
    return a + b


def test_add():
    assert add(2, 3) == 5
```

## Why teams like pytest

- less ceremony
- clearer failures
- fixtures scale better than setUp/tearDown
- easy parameterization
