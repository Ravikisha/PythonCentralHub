## Why this phase matters

Most ML projects succeed or fail **before** you train a model.

Bad preprocessing causes:

- garbage-in-garbage-out predictions
- data leakage (too-good-to-be-true results)
- models that break in production

## What you’ll be able to do after Phase 2

- handle missing values (numerical + categorical)
- encode categorical variables safely
- scale features correctly (without leakage)
- engineer simple, high-impact features
- handle imbalanced data using the right strategy
- reduce dimensionality using PCA
- split data into train/validation/test properly
