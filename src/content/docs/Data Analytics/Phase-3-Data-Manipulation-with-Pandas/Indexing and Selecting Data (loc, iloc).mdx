## Why selection matters

Most Pandas bugs come from:

- Selecting the wrong rows
- Accidentally getting a copy instead of a view
- Confusing label-based vs position-based indexing

Pandas provides two primary tools:

- `loc` → label-based
- `iloc` → position-based

## Setup example DataFrame

```python title="Sample DataFrame" showLineNumbers{1}
import pandas as pd

df = pd.DataFrame(
    {
        "name": ["Asha", "Ravi", "Meera", "Zoya"],
        "age": [23, 28, 26, 31],
        "city": ["Pune", "Delhi", "Delhi", "Pune"],
        "score": [88, 91, 77, 95],
    },
    index=["u1", "u2", "u3", "u4"],
)

print(df)
```

## Selecting columns

```python title="One column" showLineNumbers{1}
print(df["age"])  # returns a Series
```

```python title="Multiple columns" showLineNumbers{1}
print(df[["name", "score"]])
```

## Selecting rows with `loc` (labels)

```python title="loc by label" showLineNumbers{1}
print(df.loc["u2"])          # one row (Series)
print(df.loc[["u1", "u4"]]) # multiple rows (DataFrame)
```

Selecting rows + columns together:

```python title="loc rows + cols" showLineNumbers{1}
print(df.loc[["u1", "u3"], ["name", "score"]])
```

## Selecting rows with `iloc` (positions)

```python title="iloc by position" showLineNumbers{1}
print(df.iloc[0])      # first row
print(df.iloc[0:2])    # first two rows
print(df.iloc[:, 0:2]) # first two cols
```

## Boolean filtering (most common)

```python title="Filter rows" showLineNumbers{1}
filtered = df[df["city"] == "Delhi"]
print(filtered)
```

Combine conditions:

```python title="Multiple conditions" showLineNumbers{1}
filtered = df[(df["city"] == "Pune") & (df["score"] >= 90)]
print(filtered)
```

## Safer assignment after filtering

When you filter and then assign, prefer `.loc`:

```python title="Safe assignment with loc" showLineNumbers{1}
df.loc[df["city"] == "Delhi", "is_delhi"] = True

df["is_delhi"] = df["is_delhi"].fillna(False)
print(df)
```

## Note on `&` and `|`

In Pandas boolean filters:

- Use `&` for AND, `|` for OR
- Always wrap sub-conditions in parentheses

This avoids operator precedence bugs.
