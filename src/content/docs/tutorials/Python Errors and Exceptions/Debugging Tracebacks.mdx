## A repeatable debugging process

When you see an exception:

1. **Read the last line** (exception type + message)
2. **Go to the line number** where it occurred
3. Check the types/values of variables
4. Create a minimal reproduction
5. Fix and re-run

## Example traceback

```python title="bug.py" showLineNumbers{1}
def avg(nums):
    return sum(nums) / len(nums)

print(avg([]))
```

```text title="Traceback" showLineNumbers{1}
Traceback (most recent call last):
  File "bug.py", line 4, in <module>
    print(avg([]))
  File "bug.py", line 2, in avg
    return sum(nums) / len(nums)
ZeroDivisionError: division by zero
```

## Fix approach

- The list is empty, so `len(nums)` is 0.
- Decide policy: return `None`, raise a clear error, or handle with default.

```python title="fix.py" showLineNumbers{1}
def avg(nums):
    if len(nums) == 0:
        raise ValueError("avg() requires at least one number")
    return sum(nums) / len(nums)

print(avg([10, 20]))
```

## Quick tip: print types

```python title="type_debug.py" showLineNumbers{1}
value = "10"
print(type(value), value)
print(int(value))
```

## Optional: use the debugger

Python has a built-in debugger.

```python title="pdb_example.py" showLineNumbers{1}
import pdb

x = 10
pdb.set_trace()
print(x)
```

Useful commands inside the debugger:

- `n` next
- `s` step into
- `p var` print variable
- `c` continue
