## Step-by-Step Explanation

### 1. The `Event` data class
```python title="basiccalendarapp.py"
from dataclasses import dataclass, field, asdict
from datetime import datetime, date, timedelta

@dataclass
class Event:
    id: str
    title: str
    date: str               # YYYY-MM-DD
    time: str = ""          # HH:MM
    description: str = ""
    category: str = "General"
    created_at: str = field(default_factory=lambda: datetime.now().isoformat())

    def dt(self) -> datetime:
        if self.time:
            return datetime.strptime(f"{self.date} {self.time}", "%Y-%m-%d %H:%M")
        return datetime.strptime(self.date, "%Y-%m-%d")

    def is_upcoming(self, within_days: int = 7) -> bool:
        return date.today() <= self.dt().date() <= date.today() + timedelta(days=within_days)

    def to_dict(self): return asdict(self)
    @classmethod
    def from_dict(cls, d): return cls(**d)
```
Storing date and time as **ISO strings** is deliberate — JSON has no native datetime, and ISO format sorts correctly lexicographically.

### 2. The `Calendar` class
```python title="basiccalendarapp.py"
import json
from pathlib import Path

class Calendar:
    def __init__(self, filename="calendar_data.json"):
        self.path = Path(filename)
        self.events: list[Event] = []
        self.next_id = 1
        self.load()

    def load(self):
        if self.path.exists() and self.path.stat().st_size:
            data = json.loads(self.path.read_text(encoding="utf-8"))
            self.events = [Event.from_dict(d) for d in data["events"]]
            self.next_id = data.get("next_id", len(self.events) + 1)

    def save(self):
        self.path.write_text(json.dumps({
            "events": [e.to_dict() for e in self.events],
            "next_id": self.next_id,
        }, indent=2), encoding="utf-8")

    def add(self, title, date_str, time_str="", description="", category="General"):
        eid = f"EVT{self.next_id:03d}"
        self.events.append(Event(eid, title, date_str, time_str, description, category))
        self.next_id += 1
        self.save()
        return eid
```
`EVT001`, `EVT002`, `...` IDs are human-readable and survive sorting.

### 3. Date input with validation
```python title="basiccalendarapp.py"
def get_date(prompt: str) -> str:
    while True:
        raw = input(prompt + " (YYYY-MM-DD): ").strip()
        try:
            datetime.strptime(raw, "%Y-%m-%d")
            return raw
        except ValueError:
            print("Bad format. Example: 2026-01-15")
```
`strptime` *validates while it parses* — fail-early.

### 4. Monthly view with event indicators
```python title="basiccalendarapp.py"
import calendar as cal

def month_view(self, year: int, month: int) -> str:
    has_event = {e.date for e in self.events
                 if e.dt().year == year and e.dt().month == month}
    text_cal = cal.month(year, month)
    lines = text_cal.splitlines()
    out = [lines[0], lines[1]]
    for line in lines[2:]:
        new = []
        for piece in line.split():
            d = f"{year:04d}-{month:02d}-{int(piece):02d}"
            new.append(f"{piece}*" if d in has_event else piece.rjust(2))
        out.append("  ".join(new))
    return "\n".join(out)
```
`calendar.month()` does the heavy formatting; we just sprinkle `*` next to days with events.

### 5. Today / upcoming / search
```python title="basiccalendarapp.py"
def today(self):
    today_str = date.today().isoformat()
    return [e for e in self.events if e.date == today_str]

def upcoming(self, days=7):
    return sorted([e for e in self.events if e.is_upcoming(days)],
                  key=lambda e: e.dt())

def search(self, query: str):
    q = query.lower()
    return [e for e in self.events
            if q in e.title.lower() or q in e.description.lower()]
```

## Recurring Events

A real calendar must handle "every Monday at 10:00" or "the 15th of every month." Store recurrence as an extra field:
```python title="recurring.py"
@dataclass
class Event:
    # … existing fields …
    recurrence: str = ""        # "daily", "weekly", "monthly", "yearly", ""

def expand(event: Event, until: date) -> list[Event]:
    out = [event]
    start = event.dt().date()
    if event.recurrence == "daily":
        step = timedelta(days=1)
    elif event.recurrence == "weekly":
        step = timedelta(weeks=1)
    elif event.recurrence == "monthly":
        # naive month-step — use python-dateutil for correctness
        step = timedelta(days=30)
    else:
        return out
    d = start + step
    while d <= until:
        out.append(replace(event, date=d.isoformat()))
        d += step
    return out
```
For production correctness, use the **`dateutil.rrule`** library (RFC 5545 compliant) — it handles "second Tuesday of the month" and exceptions cleanly.

## ICS Export (Google / Apple Calendar Compatible)

```python title="ics.py"
def to_ics(events: list[Event]) -> str:
    lines = ["BEGIN:VCALENDAR", "VERSION:2.0", "PRODID:-//PCH//Calendar//EN"]
    for e in events:
        dt = e.dt()
        end = dt + timedelta(hours=1)
        lines += [
            "BEGIN:VEVENT",
            f"UID:{e.id}@pythoncentralhub",
            f"DTSTAMP:{datetime.utcnow().strftime('%Y%m%dT%H%M%SZ')}",
            f"DTSTART:{dt.strftime('%Y%m%dT%H%M%S')}",
            f"DTEND:{end.strftime('%Y%m%dT%H%M%S')}",
            f"SUMMARY:{e.title}",
            f"DESCRIPTION:{e.description}",
            f"CATEGORIES:{e.category}",
            "END:VEVENT",
        ]
    lines.append("END:VCALENDAR")
    return "\r\n".join(lines)
```
Save as `calendar.ics`. Drop into Google Calendar's import dialog or Apple Calendar — every event appears. **`pip install icalendar`** for a production-grade library.

## Desktop Reminders

Combine with [Simple Reminder App](./simplereminderapp). On startup, schedule one timer per upcoming event:
```python title="remind.py"
import threading
from plyer import notification

def schedule(event):
    delay = (event.dt() - datetime.now()).total_seconds()
    if delay <= 0: return
    def fire():
        notification.notify(title=event.title,
                            message=event.description or "Event now.",
                            timeout=10)
    threading.Timer(delay, fire).start()
```

## Common Mistakes

| Problem | Cause | Fix |
|---|---|---|
| `ValueError: time data '...' does not match format` | Mixed date formats | Validate with `strptime` on entry |
| Events on Feb 30 accepted | No real-date validation | `strptime` rejects them — use it |
| Timezone confusion | Naive datetimes | Use `datetime.now(timezone.utc)` consistently |
| Recurring events past year 9999 | No upper bound | Cap with a `RRULE UNTIL` or hard limit |
| Two events same minute reorder randomly | Sort key is date only | Sort by full `dt()` |
| `next_id` collisions on restart | Did not save | Persist counter alongside events |

## Why Real Calendars Are Hard

Once you go past toy versions:
- **Daylight saving** — events move an hour, sometimes split.
- **Cross-timezone scheduling** — "9 AM PT" means three different wall times across the year.
- **Rule exceptions** — "every Monday except holidays."
- **Mutable past events** — moving an instance of a recurring event without breaking others.
- **Conflict resolution** for events synced from multiple sources.
- **Notifications** that must fire even when the device is asleep.

Every major calendar app (Google, Apple, Outlook) has hundreds of engineers working on these — but the foundation is exactly the data model you just built.
